Projeto autoral em produção e evolução

Souzas Dev

Portfólio, API, painel administrativo e infraestrutura em produção.

Uma aplicação web autoral que conecta frontend sem framework, API REST em Node.js, PostgreSQL, autenticação administrativa, deploy contínuo e camadas reais de segurança.

Ver site em produção ↗ Frontend no GitHub ↗ Backend no GitHub ↗
01

Visão geral

A Souzas Dev reúne site público, formulário de contato, autenticação e painel administrativo conectados a uma API própria.

02

Desafio

Construir uma aplicação completa sem frameworks no frontend, integrando diferentes serviços sem perder organização, desempenho e segurança.

03

Solução

Frontend na Vercel, API Node.js no Render, PostgreSQL no Supabase e domínio público protegido e distribuído pela Cloudflare.

O que foi desenvolvido

Uma aplicação conectada de ponta a ponta.

A interface pública, a administração, a API e o banco de dados foram desenvolvidos como partes independentes, mas integradas em uma única experiência.

Experiência pública

  • Site institucional responsivo
  • Apresentação de serviços e projetos
  • Formulário integrado à API REST
  • Validação e retorno visual dos campos
  • Indicador de disponibilidade da aplicação
  • Case técnico com URL própria

Área administrativa

  • Login administrativo protegido
  • Autenticação baseada em JWT
  • Cookie HttpOnly, Secure e SameSite
  • Painel de mensagens recebidas
  • Busca e filtros por situação
  • Status de mensagens e exclusão controlada

Arquitetura e segurança

  • Frontend e backend separados
  • PostgreSQL utilizado em produção
  • SQLite mantido para ambiente local
  • Proteções de CORS e CSRF
  • Rate limiting em rotas sensíveis
  • Headers HTTP e CSP efetiva

Arquitetura em produção

Serviços separados, integrados e protegidos.

Cada camada possui uma responsabilidade clara, reduzindo acoplamento e permitindo evolução, manutenção e deploy independentes.

Frontend

Vercel

Hospeda e distribui o frontend estático, com deploy automático a partir do GitHub.

Backend

Render

Executa a API em Node.js publicada no subdomínio api.souzasdev.com.

Banco de dados

Supabase

Fornece o PostgreSQL utilizado para persistir mensagens e dados administrativos em produção.

Entrega e proteção

Cloudflare

Gerencia DNS, HTTPS, proxy do site público, mitigação DDoS e regras de proteção.

Segurança aplicada

Proteção distribuída em diferentes camadas.

A segurança não depende de uma única ferramenta. Navegador, aplicação, autenticação, infraestrutura e banco de dados possuem controles próprios.

01

Transporte seguro

HTTPS obrigatório, TLS moderno, HSTS e conexão criptografada entre os serviços.

02

Proteção no navegador

Content Security Policy, bloqueio de frames, proteção contra MIME sniffing e políticas de origem.

03

Autenticação

JWT em cookie protegido, defesa contra CSRF, CORS restrito e limite de tentativas.

04

Perímetro

Cloudflare no site público, mitigação DDoS e desafio gerenciado para a área administrativa.

Evolução

De exercício local a sistema publicado.

O projeto começou como uma prática de HTML, CSS e JavaScript e evoluiu para uma arquitetura composta por frontend, API, autenticação, banco de dados e painel administrativo.

A transição do SQLite local para PostgreSQL em produção, a separação dos deploys e o endurecimento das configurações de segurança transformaram o portfólio em uma aplicação real e continuamente aprimorada.
